Android注解式findViewByID以及注解的几种常见用法

发表于 2016-05-31   |   分类于 Androd   |  

从findViewByID说开去

在Android开发中，界面的编写少不了要使用findviewbyid方法来定位一个控件。麻烦的一是几乎每个控件都需要”findviewbyid”一次。。冗余，不优雅；其次是在findviewbyid的时候，参数id往往难以记住。我们希望能够在申明这个控件的时候就能够一劳永逸的完成控件与id的绑定关系。好在我们可以使用注解来做。先看看具体做法。

基本思路

首先，我们在申明一个的view控件的语句上方，加入注解。我们知道的是，可以以某种方法来获取这个注解中的值，这个值可以事先设置为该控件的id值。

其次，在获取到id之后，就只需要找一个地方来运行findviewbyid，然后将find到的view控件赋值给之前的view变量即可。

实际上我们在activity中调用的findviewbyid最终都是以view.findviewbyid的形式出现的，于是我们只要能够得到activity布局的父容器控件，则就可以轻松的通过view.findviewbyid找到其下所有的子view。

Resource Type Annotations

这一类注解用于Android中的资源类型区分，比如说在Android中，xml中的String和view类型的变量的id都是int型的，如果他们是参数的话，如果参数类型只有一个int来标识，使用的时候可能会给人带来困惑，到底是传一个String的id进去还是传一个view的id进去？而Resource Type Annotations就是用来做这类区别的，它有如下几种：@StringRes, @DrawableRes, @ColorRes, @InterpolatorRes，@AnyRes。
我们只需要在对应的资源id变量之前使用对应的注解，编译器就会自动检查调用时传入的参数是否符合规定。

IntDef/StringDef: Typedef Annotations

类型注解（IntDef/StringDef: Typedef Annotations）跟上面的有所不同，它更加类似于Java中的Enum(枚举类型)，而按照google文档的说法，Enum在Android中不推荐使用，因为它比一般类型要占用多一倍的内存。具体用法是：使用IntDef或StringDef先定义一个包含几个具体值的集合，并将其作为某个注解类型(@interface)的元注解，然后使用该注解类型去标识对应的参数类型，即可让不在集合内的传参变为非法。也就变相的达到的类似于Enum的功能，代码如下：
